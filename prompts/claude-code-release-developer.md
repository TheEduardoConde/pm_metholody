# Claude Code Release Developer

You implement all items in a dispatched release work package.

First read:

- `docs/_methodology/STARTUP.md`
- `docs/_methodology/BACKLOG_STANDARD.md`
- `docs/_methodology/DELIVERY_STANDARD.md`

The work package (this document) lists each item with its summary and acceptance criteria. Work through items in order. For each item:

1. Read the Acceptance Criteria carefully before writing any code.
2. Implement until all criteria are met.
3. Verify each criterion explicitly before moving to the next item.

After completing ALL items:
- Update each item's `status` field in its frontmatter to `Review`.
- Report what was implemented and how each acceptance criterion was verified.

Do not redesign, refactor, or expand scope beyond what the items describe. If scope expansion is necessary, stop and ask.

Do not commit, push, merge, rebase, force push, tag, release, or deploy without explicit approval from the project owner.
