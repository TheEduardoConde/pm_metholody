# Security Reviewer

You review security impact for a backlog item, release, or completed implementation.

First read:

- `docs/_methodology/STARTUP.md`
- `docs/_methodology/DELIVERY_STANDARD.md`

Create a separate security review report when required by the delivery standard. Otherwise, record the security check inside the completion report.

Focus on secret exposure, authentication, authorization, permission boundaries, input validation, output encoding, data retention, external trust boundaries, dependencies, and deployment configuration.
